What software do I need to learn to be a UX designer?
This is a really common question that many people new to the industry lead off with.
Unfortunately, the long answer is a huge it depends.
The short answer is - refer to the 2020 UX tools survey results
There is no one-size-fit-all tool.
UX is is a big umbrella term that covers many processes, each with their unique workflows and goals. Generally when people talk about UX design tools, they are referring to either one of the following two:
1.
UI design tools
2.
Prototyping tools
In the past, Sketch would be considered a UI design software and InVision would be considered a prototyping software. Figma is the one topping all the charts in 2020. It is seen as a hybrid solution which is capable of both designing UIs and creating prototypes to test with.
A tool for every situation
Of course, when you actually start diving into the industry, you will see that there are so many more "types" of UX tools. The tools survey does a pretty great job at breaking it down into the general categories of:
1.
Brainstorming - Tools like Miro that you can run workshops with
2.
User Flows - Visualization tools like Whimsical to organize ideas
3.
UI Design - Tools like Sketch or even Adobe Illustrator could fit in here
4.
Prototyping - Tools like Figma or more heavy duty ones like Axure
5.
User Testing - Could be as simple as click testing through Helio
6.
Analytics - Google analytics works, but there are fancier solutions such as Fullstory
7.
Versioning - Abstract is a popular tool for facilitating collaboration in a way similar to Git
8.
Documentation/Project Management - Atlassian is popular, but there are many out there
9.
Handoff - Mostly integrated into other tools, but some apps focus on this, such as Avocode
10.
Design Systems - This depends heavily on your UI toolset, but often includes some "bridge" with developers. Storybook is a popular one for sharing UI components
Even with these 10 categories, I only just list one each that comes to mind. There are many options to choose from and they are all valid for different use cases.
Depending on what your role in a project is, you will be required to utilize different tools. For example, if you are a UX researcher, you might be able to get away with focusing on tools (1-Brainstorming, 2-User Flows, 5-User Testing, 6-Analytics, 8-Documentation/PM). In this specific scenario, it might not be that important for you to learn Figma because there will likely be a UX Designer who will be creating the UI mockups and prototypes that you will use for usability testing.
If your role is more of a "jack-of-all-trades" designer, you might be expected to have rudimentary knowledge in enough tools to cover all the different situations you encounter.
How do I decide what tool to learn?
At the end of the day, you should learn whatever tool the job you want requires you to know. Take a look at opportunities you are interested in and see what tools they are looking for.
Most design teams already have a tool of choice and have some sort of process already established. This could be as fancy and fleshed out as a documented design system or as casual as a file that contains all the different buttons used in the product and a Google Doc.
The most important thing for many teams is to have a scalable process that has enough redundancies to produce good designs regardless of any employee churn that might occur. This means that you, as a new hire, should have access to the tools and documentation that can result in the same consistent quality of deliverables as any other designer on the team. Therefore, there tends to be little wiggle room to move to a whole new toolset without a serious discussion with everyone involved in the design process.
(So just for that point above, if you are the sole designer of a product, maybe it is easy to switch up the tools because you don't have that same overhead).
But what if I learned X and the Job requires Y? Etc...
Just learn the tools they want.
It is honestly not that hard to learn most of the really popular tools in the industry (Sketch, XD, Figma) and they are also quite similar... Download a free trial and watch some YouTube tutorials. If the job requires a specific software, then it is probably in your best interest to give the software a test run first.
For some of the slightly more complex and niche ones like Axure, Protopie, Origami, Bubble etc.. you might want to take more time to get comfortable with them as the learning curve is a bit steeper.
For the most niche jobs such as in VR/AR, you might be expected to know very advanced tools like Unity. If that is the case, you should be asking yourself if you stand a chance at the position if they are asking for an advanced tool that you don't know. If you still want that job, you better start watching tutorials ASAP!
It really does depend eh?
There are so many factors when considering design tools. I would be highly suspicious of anyone who claims one is flat-out superior to another. This is why there are so many options out there.
Sure, there are some tools that might be better for:
•
Responsive mobile apps vs. native apps
•
Building prototypes that take advantage of native device inputs (voice, location, gyro, etc...)
•
Collaborating with a large team
•
Working with real data inputs and outputs
•
Building scalable design systems
•
Taking advantage of conditional logic to power prototypes
•
Keeping things simple with less visual clutter
•
Handling complex non-linear workflows
•
And many, many more...
This is just the tip of the iceberg. Depending on your team structure, industry, timelines, project complexity, intended users, available resources etc... what an ideal workflow and toolset are varies drastically from project to project.
In conclusion...
I circle back to my short answer. Refer to the 2020 UX tools survey results for the data on what tools are most popular. It's a good place to start.
Add in any specific tools your dream job mentions in their requirements.
Remember, tools are tools and your value as a UX designer is not to "use tools". It is to deliver results through the most effective means, which happens to include using some tools
